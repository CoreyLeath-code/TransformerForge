# Security Policy

## Supported versions

Security fixes are applied to the latest release and the `main` branch.

## Reporting a vulnerability

Do not disclose suspected vulnerabilities in a public issue. Use GitHub private vulnerability reporting when available, or contact the repository owner privately with the affected component, reproduction steps, impact assessment, and suggested mitigation.

## Automated controls

TransformerForge uses CodeQL, Gitleaks, Trivy, pip-audit, Dependabot, CycloneDX SBOM generation, multi-version tests, container health checks, and release-readiness validation.
